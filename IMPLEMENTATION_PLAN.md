# План імплементації AquaSync (Citizen PWA)

Цей документ описує покроковий план створення мобільного додатку **AquaSync (Citizen PWA)** згідно зі специфікацією в `project-frontend-admin.md`. Хоча файл опису названий "admin", його вміст фокусується на додатку для громадян, тому ми реалізуємо його у папці `frontend-citizen`.

## 🎨 Дизайн та Візуальна концепція
- **Стиль:** "Eco-Tech Premium" — поєднання екологічної тематики з високотехнологічним виглядом.
- **Елементи:** Glassmorphism (ефект матового скла), плавні CSS-анімації, градієнти "Oceanic Blues" та "Deep Forest Greens".
- **Шрифти:** Використання сучасних шрифтів (Outfit або Inter) через Google Fonts для преміального вигляду.

## 🛠 Технологічний стек
- **Frontend:** React.js (Vite) — для швидкості та легкості.
- **PWA:** `vite-plugin-pwa` — забезпечує офлайн-роботу та встановлення на робочий стіл (установка "в один клік").
- **Стилізація:** Vanilla CSS з використанням сучасних CSS-змінних та Grid/Flexbox.
- **Камера:** `react-webcam` — захоплення фото в реальному часі без можливості вибору з галереї (Anti-fraud захист).
- **Геолокація:** Native Geolocation API для реалізації "Радара Галілео".

## 🚀 План розробки

### Етап 1: Фундамент та Дизайн-система (Сьогодні)
1. **Ініціалізація:** Налаштування проєкту в `frontend-citizen/`.
2. **Дизайн-система:** Створення `index.css` з усіма змінними (кольори, тіні, розмиття скла).
3. **PWA Config:** Налаштування manifest та Service Worker для кешування та офлайн-режиму.

### Етап 2: Ядро — "Action Camera" (Екран 2)
1. **Камера:** Повноекранний UI для зйомки.
2. **Радар точності Galileo:** 
   - Візуальний індикатор точності GPS.
   - Логіка блокування кнопки спуску до досягнення точності < 5м.
3. **Хуки стану:** Відстеження координат у реальному часі.

### Етап 3: Потік створення звіту (Екран 3)
1. **Прев'ю:** Фото з накладеними метаданими (Watermark).
2. **Тагування:** Інтерактивні кнопки для вибору типу забруднення.
3. **Сховище:** Реалізація черги відправки (Background Sync), щоб додаток працював навіть у "мертвих зонах" покриття.

### Етап 4: Особистий кабінет та Гейміфікація (Екрани 4, 5)
1. **Dashboard:** Метрики (бали, площа охопленої території).
2. **Беджі:** Реалізація системи нагород (кольорові vs сірі іконки).
3. **Історія:** Список звітів з яскравими статус-барами (Processing -> Verified -> Alert -> Resolved).

### Етап 5: Онбординг та Соціум (Екрани 1, 6)
1. **Onboarding:** Слайд-шоу, що пояснює місію.
2. **Auth:** Спрощена реєстрація.
3. **Community Map:** Heatmap активності користувачів через Leaflet.js.
